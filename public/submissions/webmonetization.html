Introduction

The Web Monetization API is an open JavaScript API that allows websites to accept micropayments using web monetization providers. This tutorial will walk you through the process of integrating the Web Monetization API into your website, setting up event listeners, and tracking monetization progress.

Setup

To start using Web Monetization, you need to add a payment pointer to your website. The payment pointer is included in the <meta> tag within your HTML document:

<meta name="monetization" content="$your-payment-pointer">
Note: Make sure to replace $your-payment-pointer with your actual payment pointer from your web monetization provider.

Usage Example

Once you've added the payment pointer, you can interact with the Web Monetization API through JavaScript. Here's a basic example of how to check if monetization is enabled and respond to the events:

if (document.monetization) { console.log('Web Monetization is supported!'); document.monetization.addEventListener('monetizationstart', function(event) { console.log('Monetization has started!'); // Handle the start of monetization here }); document.monetization.addEventListener('monetizationprogress', function(event) { console.log(`Amount: ${event.detail.amount}`); console.log(`Asset Code: ${event.detail.assetCode}`); console.log(`Scale: ${event.detail.assetScale}`); // Track payment progress here }); } else { console.log('Web Monetization is not supported on this browser.'); }

Web Monetization Events

The Web Monetization API emits the following events you can listen for:

  • monetizationstart: Fired when monetization starts (e.g., when a user visits a page and starts streaming payments).
  • monetizationprogress: Fired periodically as payments are made, providing details about the amount, asset code, and scale.
  • monetizationstop: Fired when monetization stops (e.g., when the user navigates away or the payment provider stops streaming).
